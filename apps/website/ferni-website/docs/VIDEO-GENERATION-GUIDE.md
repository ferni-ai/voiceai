# 🎬 Ferni Video Generation Guide
## Using Google Veo 3 for Feature Videos

---

## Quick Start

### 1. Access Veo 3
Go to: **https://labs.google.com/videotools**

### 2. Copy a Prompt
Use prompts from `prompts/VEO3-PROMPTS.txt`

### 3. Generate & Download
- Select 4K or 8K resolution
- Generate video
- Download when ready

### 4. Process Video
```bash
cd apps/website/ferni-website

# Optimize for web (creates WebM + MP4)
node scripts/optimize-assets.js --type=videos
```

## Integration Guide

### Adding Video to Homepage

**HTML Structure:**
```html
<div class="feature-video">
  <video 
    autoplay 
    muted 
    loop 
    playsinline
    poster="images/hero-fallback.jpg"
    data-src="videos/hero-meadow.webm"
  >
    <source src="videos/hero-meadow.webm" type="video/webm">
    <source src="videos/hero-meadow.mp4" type="video/mp4">
  </video>
  <div class="feature-video-fallback">
    <!-- CSS animation fallback -->
  </div>
</div>
```

**Required CSS:**
Include `css/feature-videos.css` in your page.

**Required JS:**
Include `js/feature-videos.js` for Intersection Observer autoplay.

## Quality Settings

### Veo 3 Generation
- **Resolution:** 4K minimum, 8K preferred
- **Duration:** 10-15 seconds for loops
- **Style:** Cinematic, slow movement

### FFmpeg Optimization (auto via script)
- **WebM:** VP9 codec, CRF 30
- **MP4:** H.264, CRF 28
- **Max Width:** 1920px for web, 1280px for mobile

## Troubleshooting

### Video doesn't autoplay
- Ensure `muted` attribute is present
- Check `playsinline` for mobile Safari
- Verify Intersection Observer is loaded

### Video too large
```bash
# Re-optimize with lower quality
ffmpeg -i input.mp4 -c:v libvpx-vp9 -crf 35 -b:v 0 output.webm
```

### Colors look wrong
- Check color grading in Veo prompt
- Avoid "cool blues" or neon colors
- Use specific hex codes in prompt
