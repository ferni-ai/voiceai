# 1. Philosophy

## The Ferni Illustration Approach

Our illustrations embody the same principles as our product: **warm, grounded, human, and thoughtful**. They should feel like they were drawn by a friend, not generated by a computer.

### Core Principles

| Principle | Description |
|-----------|-------------|
| **Organic** | Slightly imperfect, hand-drawn quality |
| **Warm** | Earth tones, gentle curves, approachable |
| **Minimal** | Only essential elements, maximum meaning |
| **Universal** | Abstract enough to be inclusive |
| **Serene** | Calm, never chaotic or busy |

### What Our Illustrations Are NOT

❌ Hyper-realistic or photographic  
❌ Cartoon-y or childish  
❌ Corporate clip-art  
❌ 3D renders or gradients  
❌ Busy or cluttered  
❌ Generic tech/startup aesthetic

# 2. Style Attributes

## 2.1 Line Work

### Stroke Characteristics

| Attribute | Specification |
|-----------|---------------|
| **Weight** | 1.5px base (scales with illustration size) |
| **Caps** | Round |
| **Joins** | Round |
| **Style** | Continuous where possible, never dotted |
| **Character** | Slightly organic, not perfectly straight |

### Line Hierarchy

```
Primary Lines:   1.5px - Main shapes, focal elements
Secondary Lines: 1px   - Details, textures
Accent Lines:    2px   - Emphasis, important elements
```

---

## 2.2 Color Usage

### Primary Illustration Palette

| Role | Light Theme | Dark Theme | Usage |
|------|-------------|------------|-------|
| **Line Color** | Natural Ink (#2C2520) | Paper Cream (#FAF8F5) | All strokes |
| **Fill (Primary)** | Persona Primary | Persona Primary | Focal elements |
| **Fill (Secondary)** | rgba(persona, 0.15) | rgba(persona, 0.2) | Background shapes |
| **Accent** | Warm Amber (#C4A265) | Warm Amber | Highlights, emphasis |

### Color Application Rules

1. **Maximum 3 colors** per illustration
2. **One fill color** maximum (persona or accent)
3. **Line color is always primary** — fills are accents
4. **Transparency creates depth** — layer with opacity

---

## 2.3 Shapes

### Shape Language

| Shape | Meaning | Usage |
|-------|---------|-------|
| **Circles** | Wholeness, unity, connection | Personas, avatars, orbs |
| **Soft Rectangles** | Stability, grounding | Frames, containers |
| **Organic Blobs** | Life, growth, nature | Backgrounds, accents |
| **Arcs** | Journey, progress, flow | Paths, connections |
| **Dots** | Points, moments, particles | Decoration, rhythm |

### Corner Radius

All corners should be rounded:
- Small elements: 4px radius
- Medium elements: 8px radius
- Large elements: 12-24px radius
- Never sharp 90° corners in illustrations

---

## 2.4 Composition

### Grid

All illustrations should be composed on an underlying grid:

```
Small (64×64):    8×8 grid, 8px units
Medium (128×128): 16×16 grid, 8px units
Large (256×256):  32×32 grid, 8px units
Hero (512×512):   64×64 grid, 8px units
```

### Balance

- **Asymmetric but balanced** — Not centered, but visually weighted
- **Breathing room** — 10-15% padding minimum
- **Focal point** — One clear area of attention
- **Flow** — Eye should move naturally through illustration

# 4. Persona Representations

## 4.1 The Non-Face Approach

Ferni personas are represented **abstractly**, not as literal faces or characters. This allows users to project their own imagination.

### Representation Options

#### Option A: Colored Orbs (Current)
Simple glowing circles with persona colors.

```svg
<!-- Ferni Orb -->
<circle cx="40" cy="40" r="32" fill="#4a6741" />
<circle cx="40" cy="40" r="36" fill="none" stroke="#4a6741" 
        stroke-opacity="0.3" stroke-width="8" />
```

#### Option B: Abstract Presence (Recommended)
Organic shapes that suggest presence without defining it.

```
Ferni:    Grounded circle with subtle glow
Jack:     Deeper shape, like a stone
Peter:    Shape with rays/lines suggesting curiosity
Alex:     Flowing, connective shape
Maya:     Structured but organic grid
Jordan:   Light, sparkly shape
Nayan:    Encompassing, integrative shape
```

#### Option C: Symbolic Marks
Simple marks that evoke the persona's essence.

```
Ferni:    ○  Simple circle (wholeness)
Jack:     ◯  Heavier ring (wisdom/depth)
Peter:    ✧  Star-like (curiosity)
Alex:     ∿  Wave (communication)
Maya:     ⊞  Grid (structure)
Jordan:   ✦  Sparkle (celebration)
Nayan:    ◎  Concentric (integration)
```

# 5. Empty States

## 5.1 Empty State Philosophy

Empty states are opportunities, not failures. They should:
- Feel inviting, not empty
- Suggest potential, not absence
- Guide action, not frustrate

## 5.2 Empty State Library

### No Conversations Yet

**Illustration:** Two abstract shapes with space between them, potential connection indicated by dotted arc.

**Visual Elements:**
- Two organic circles
- Dotted line suggesting potential connection
- Warm, inviting colors
- Upward/forward orientation

**Composition:**
```
┌─────────────────────────┐
│                         │
│    ○ ··········· ○     │
│                         │
│  "Your story starts     │
│   with 'hello'"         │
│                         │
│    [ Start Talking ]    │
│                         │
└─────────────────────────┘
```

### No Team Members Unlocked

**Illustration:** Central presence (Ferni) with faded shapes around suggesting waiting team.

**Visual Elements:**
- Ferni orb prominent
- Ghosted shapes suggesting others
- Warm, welcoming composition
- Growth/time suggestion

### No Progress Data

**Illustration:** Gentle upward path with first step indicated.

**Visual Elements:**
- Rising arc/path
- First step marked
- Destination suggested, not defined
- Journey metaphor

### Search No Results

**Illustration:** Magnifying glass with gentle shrug (abstract).

**Visual Elements:**
- Search circle
- Subtle "?" or exploration marks
- Not sad—curious
- Invitation to try again

# 6. Error States

## 6.1 Error State Philosophy

Errors should feel like temporary setbacks, not failures. They should:
- Acknowledge without alarming
- Suggest solutions
- Maintain brand warmth

## 6.2 Error State Library

### Connection Lost

**Illustration:** Two shapes with broken line between, but shapes remain intact.

**Message:** "Taking a breath. We'll reconnect soon."

**Visual Elements:**
- Two personas (abstract)
- Dashed/broken connection line
- Both shapes stable
- Reassuring composition

### Something Went Wrong

**Illustration:** Puzzle piece slightly askew, hand (abstract) reaching to fix.

**Message:** "That didn't work how we expected. Mind trying again?"

**Visual Elements:**
- Simple geometric puzzle
- One piece slightly off
- Suggestion of correction
- Warm, not clinical

### Offline

**Illustration:** Serene pause state, like a held breath.

**Message:** "We're offline right now. We'll be here when you're back."

**Visual Elements:**
- Calm, waiting posture
- Horizontal composition (stability)
- Muted but warm colors
- Zen garden inspiration

### Rate Limited

**Illustration:** Gentle hourglass or clock with "soon" indication.

**Message:** "We're popular today! Give us a moment."

**Visual Elements:**
- Time indication without urgency
- Patience metaphor
- Warm amber accents
- Not frustrating

# 7. Celebration Illustrations

## 7.1 Celebration Philosophy

Celebrations should feel earned and genuine, never performative. They should:
- Match the magnitude of the achievement
- Feel warm, not corporate
- Be memorable but not overwhelming

## 7.2 Celebration Library

### Small Win

**Illustration:** Single sparkle or gentle pulse

**Usage:** Daily habits, small progress

**Visual Elements:**
- Simple sparkle shape
- Persona color
- Subtle, not overwhelming
- Quick animation potential

### Streak Achievement

**Illustration:** Rising dots forming upward pattern

**Usage:** Consistency milestones (3, 5, 7, 21, 30 days)

**Visual Elements:**
- Sequential dots
- Ascending pattern
- Accumulation feeling
- Warm amber accents

### Breakthrough Moment

**Illustration:** Expanding circle with rays

**Usage:** Growth reflections, significant insights

**Visual Elements:**
- Central expansion
- Radiating lines
- Breakthrough feeling
- Full persona glow

### Relationship Milestone

**Illustration:** Two shapes becoming closer, more intertwined

**Usage:** Stage upgrades (Building Trust → Established, etc.)

**Visual Elements:**
- Progression sequence
- Deepening connection
- Journey completion feeling
- Multi-color (Ferni + user)

### Team Unlock

**Illustration:** New shape appearing, joining constellation

**Usage:** New persona unlocked

**Visual Elements:**
- New persona orb appearing
- Connection to existing team
- Welcome/arrival feeling
- Persona-specific color

# 10. Technical Specifications

## 10.1 File Formats

| Format | Usage | Specifications |
|--------|-------|----------------|
| **SVG** | Primary, web | Optimized, accessible |
| **PNG** | Fallback | 2x resolution, transparent |
| **PDF** | Print | Vector, CMYK |

## 10.2 SVG Guidelines

```xml
<!-- Required attributes -->
<svg 
  xmlns="http://www.w3.org/2000/svg" 
  viewBox="0 0 64 64"
  width="64" 
  height="64"
  role="img"
  aria-label="Description of illustration"
>
  <!-- Use currentColor for themeable elements -->
  <path stroke="currentColor" fill="none" ... />
  
  <!-- Use CSS variables for persona colors -->
  <circle fill="var(--persona-primary, #4a6741)" ... />
</svg>
```

## 10.3 Naming Convention

```
ferni-[type]-[name][-variant].svg

Examples:
ferni-empty-no-conversations.svg
ferni-error-connection-lost.svg
ferni-celebration-streak-7.svg
ferni-persona-ferni-orb.svg
```

## 10.4 Export Sizes

| Usage | Sizes Required |
|-------|----------------|
| **Spot** | 64, 128 (2x) |
| **Feature** | 128, 256 (2x) |
| **Hero** | 256, 512 (2x) |
| **Icons** | 16, 24, 32 |

# 11. Do's and Don'ts

## ✅ DO

- Keep it simple—remove until you can't
- Use organic, slightly imperfect lines
- Stay within the color palette
- Maintain breathing room
- Make illustrations accessible (labels, contrast)
- Test across themes (light/dark)
- Animate sparingly, purposefully

## ❌ DON'T

- Add details for the sake of detail
- Use complex gradients or shadows
- Create literal human faces
- Make illustrations too busy
- Use colors outside the system
- Forget accessibility
- Animate everything
