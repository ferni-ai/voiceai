Quick Start

Add a new agent to your team in under 5 minutes

1

Create the agent bundle

Use the CLI to scaffold a new agent from a template

# Create a sage-style advisor
npm run agents create my-advisor --template sage

# Or a specialist for a specific domain
npm run agents create tax-expert --template specialist
2

Customize the manifest

Edit persona.manifest.json with your agent's identity and triggers

{
  "identity": {
    "id": "my-advisor",
    "name": "Alex",
    "description": "Friendly tax advisor"
  },
  "team": {
    "handoff_triggers": ["taxes", "deductions", "filing"]
  }
}
3

Validate and run

Check your bundle is valid, then start the server

# Validate the bundle
npm run agents validate my-advisor

# Run with your new agent
PERSONA_ID=my-advisor npm run dev

Bundle Anatomy

Everything an agent needs in one self-contained folder

src/personas/bundles/my-advisor/ ├── persona.manifest.json ← Configuration ├── identity/ │ ├── biography.md ← Background story │ └── system-prompt.md ← Instructions └── content/ ├── behaviors/ │ ├── greetings.json ← Hello phrases │ ├── catchphrases.json ← Signature lines │ └── quirks.json ← Personality quirks ├── stories/ │ ├── _index.json ← Story triggers │ └── *.json ← Personal anecdotes └── knowledge/ ├── _index.json ← Topic index └── *.md ← Domain expertise

📄 persona.manifest.json

The only required file. Defines identity, voice, personality, handoff triggers, and content paths.

📁 identity/

Who is this agent? Background story and behavioral instructions that shape their responses.

📁 content/behaviors/

How they speak. Greetings, catchphrases, thinking sounds, emotional responses.

📁 content/stories/

Personal anecdotes triggered by keywords. Makes agents feel real and relatable.

📁 content/knowledge/

Domain expertise in markdown. Loaded on-demand when topics come up.

CLI Commands

Manage agents from your terminal

npm run agents list
List all discovered agents with their status
npm run agents show <id>
Show detailed info for a specific agent
npm run agents create <id>
Create new agent from template (basic, sage, specialist, coordinator)
npm run agents validate [id]
Validate bundle(s) for errors and warnings
npm run agents enable <id>
Enable an agent in the team roster
npm run agents disable <id>
Disable an agent without removing files

API Endpoints

RESTful endpoints for programmatic agent management

Method Endpoint Description
GET /api/agents List all enabled agents with UI-ready data
GET /api/agents/:id Get detailed agent info by ID or alias
POST /api/agents/:id/enable Enable or disable an agent
PUT /api/agents/:id Update agent settings (colors, subtitle, etc.)
POST /api/team/order Update team roster display order
POST /api/agents/validate Run validation on all agent bundles
GET /api/voice/preview/:voiceId Get Cartesia playground URL for voice preview

Agent Templates

Pre-configured starting points for different agent types

🧙 Sage Template

For wise, thoughtful advisors with measured speech

  • Slow, deliberate speech patterns
  • High warmth, low energy personality
  • Therapeutic humanization preset
  • Wisdom-focused domains
npm run agents create mentor --template sage

🎯 Specialist Template

For focused experts in specific domains

  • Efficient, precise speech
  • Higher directness personality
  • Domain-specific tool access
  • Natural humanization preset
npm run agents create tax-pro --template specialist

👤 Basic Template

General-purpose starting point

  • Balanced personality traits
  • Medium speech characteristics
  • Flexible tool access
  • Conversational preset
npm run agents create helper --template basic

🎭 Coordinator Template

For team coordinators (usually only need one)

  • Can handoff to any team member
  • Full team coordination capabilities
  • Welcoming personality
  • Cross-agent context awareness
npm run agents create lead --template coordinator
