# Cross-Domain Conversation Journeys

This document outlines designed conversation flows that weave through multiple deep engagement domains for meaningful, transformative conversations.

## Overview

The deep engagement domains are most powerful when used together. These journeys show how to create multi-domain conversations that address the full complexity of human experience.

---

## Journey 1: Life Review

**Purpose**: Help someone reflect on their life, capture their story, and consider their legacy.

**Domains Used**: `stories` → `relationships` → `meaning` → `dreams`

**Flow**:

1. **Opening** (stories)
   - Use `captureLifeStory` with focus on "defining-moments"
   - Let them share key turning points

2. **Relationship Exploration** (relationships)
   - Use `mapRelationshipCircles` to see their people
   - Use `expressGratitude` to acknowledge key relationships
   - Explore any `identifyNeglectedRelationships`

3. **Meaning Making** (meaning)
   - Use `clarifyValues` to identify core values
   - Use `findNarrativeThread` to see patterns and themes
   - Use `alignActionsWithPurpose` to check alignment

4. **Future Orientation** (dreams)
   - Use `futureSelf` to connect with who they want to become
   - Use `defineLegacy` to articulate what they want to leave behind
   - Use `bucketList` to surface unfulfilled aspirations

**Ideal For**: Milestone birthdays, retirement, life transitions, annual reflections

---

## Journey 2: Grief Integration

**Purpose**: Walk alongside someone processing loss toward gradual integration.

**Domains Used**: `grief` → `presence` → `stories` → `meaning`

**Flow**:

1. **Holding Space** (grief + presence)
   - Use `processGrief` to meet them where they are
   - Use `groundInBody` when emotions are overwhelming
   - Use `companionInGrief` for simple presence
   - Use `navigateGriefWave` during intense moments

2. **Honoring Loss** (grief + stories)
   - Use `rememberLoved` to celebrate who they lost
   - Use `captureLifeStory` focused on the relationship
   - Use `acknowledgeLoss` for losses others don't recognize

3. **Finding Meaning** (meaning)
   - Only when ready - use `findMeaningInSuffering`
   - Use `sitWithBigQuestion` for existential questions
   - Use `exploreMortality` for mortality awareness

4. **Integration** (grief)
   - Use `navigateTransition` for life changes
   - Use `embraceNewIdentity` for who they're becoming
   - Use `anniversarySupport` for ongoing support

**Ideal For**: Death of loved ones, divorce, major life losses

---

## Journey 3: Purpose Discovery

**Purpose**: Help someone find or reconnect with their sense of purpose.

**Domains Used**: `meaning` → `curiosity` → `dreams` → `relationships`

**Flow**:

1. **Current State Assessment** (meaning)
   - Use `explorePurpose` with current state "searching" or "lost"
   - Use `clarifyValues` to identify what matters

2. **Curiosity Exploration** (curiosity)
   - Use `experienceWonder` to reawaken sense of possibility
   - Use `intellectualExploration` to follow fascinations
   - Use `whatIf` to play with possibilities

3. **Dream Recovery** (dreams)
   - Use `reconnectWithDreams` to find forgotten aspirations
   - Use `exploreDream` to understand what they really want
   - Use `playWithPossibility` to remove constraints

4. **Relational Context** (relationships)
   - Use `mapRelationshipCircles` to see their support system
   - Consider: Who supports this purpose? Who might resist?
   - Use `suggestConnectionAction` for needed connections

5. **Alignment** (meaning)
   - Use `alignActionsWithPurpose` to create action plan
   - Use `valueConflictResolution` for any value tensions

**Ideal For**: Career transitions, midlife, feeling stuck or unfulfilled

## Journey 4: Self-Compassion Development

**Purpose**: Build a healthier relationship with oneself.

**Domains Used**: `self-compassion` → `vulnerability` → `presence` → `play`

**Flow**:

1. **Inner Critic Awareness** (self-compassion)
   - Use `noticeInnerCritic` to identify harsh self-talk
   - Use `reframeInnerCritic` to transform messages
   - Use `speakToYourselfAsAFriend` for practice

2. **Shame Exploration** (vulnerability)
   - Use `createSafeSpace` for sensitive exploration
   - Use `exploreShame` for specific shame points
   - Use `processShame` for healing work

3. **Self-Forgiveness** (vulnerability + self-compassion)
   - Use `practiceSelfForgiveness` for specific regrets
   - Use `embraceImperfection` for acceptance
   - Use `enoughness` for inherent worthiness

4. **Grounding & Integration** (presence)
   - Use `groundInBody` for embodiment
   - Use `breatheWithMe` for regulation
   - Use `noticeThisMoment` for present focus

5. **Lightness & Play** (play)
   - Use `givePermissionToPlay` for joy permission
   - Use `cultivatePlayfulness` for rebuilding fun
   - Use `celebrateYourself` for acknowledgment

**Ideal For**: Perfectionism, low self-worth, harsh self-judgment, burnout recovery

## Journey 5: Relationship Deepening

**Purpose**: Deepen a specific relationship or improve relational health overall.

**Domains Used**: `relationships` → `vulnerability` → `self-compassion` → `meaning`

**Flow**:

1. **Relationship Assessment** (relationships)
   - Use `assessRelationshipHealth` for specific relationship
   - Use `mapRelationshipCircles` for overall picture
   - Identify key areas for growth

2. **Repair & Communication** (relationships)
   - Use `navigateConflict` for existing tensions
   - Use `prepareForDifficultConversation` for hard talks
   - Use `craftApology` if repair is needed
   - Use `expressGratitude` for appreciation

3. **Vulnerability Work** (vulnerability)
   - Use `practiceTruth` for things unsaid
   - Use `revealHiddenSelf` for deeper sharing
   - Use `createSafeSpace` for sensitive topics

4. **Self-Work** (self-compassion)
   - Use `setRelationshipBoundary` for needed limits
   - Use `practiceSelfKindness` for relationship stress
   - Recognize your own patterns

5. **Values Alignment** (meaning)
   - Use `clarifyValues` to understand relational values
   - Consider: Does this relationship align with who I want to be?
   - Use `valueConflictResolution` if needed

**Ideal For**: Marriage/partnership growth, family reconciliation, friendship deepening

## Journey 6: Transition Navigation

**Purpose**: Navigate a major life transition with intention.

**Domains Used**: `grief` → `stories` → `meaning` → `dreams` → `presence`

**Flow**:

1. **Honoring What's Ending** (grief)
   - Use `processEnding` to acknowledge the transition
   - Use `acknowledgeLoss` for what's being left behind
   - Allow grief for the previous chapter

2. **Story Capture** (stories)
   - Use `exploreLifeChapter` to close the old chapter
   - Use `findNarrativeThread` to see the story arc
   - Use `rewriteStory` if needed for reframing

3. **Identity Work** (grief + meaning)
   - Use `embraceNewIdentity` for who you're becoming
   - Use `clarifyValues` for what stays constant
   - Use `sitWithBigQuestion` for "who am I now?"

4. **Future Visioning** (dreams)
   - Use `futureSelf` to connect with future you
   - Use `playWithPossibility` to imagine options
   - Use `exploreDream` for emerging desires

5. **Grounding** (presence)
   - Use `returnToPresent` when overwhelmed
   - Use `groundInBody` for stability
   - Use `protectPresence` for managing anxiety

**Ideal For**: Career change, retirement, divorce, relocation, empty nest, major health change

## Using These Journeys

### Guidelines

1. **Don't force the sequence**. These are templates, not scripts. Follow the person.

2. **Stay in one domain as long as needed**. Don't rush to the next stage.

3. **Use tools sparingly**. Not every turn needs a tool. Presence often matters more.

4. **Circle back**. Journeys aren't linear. You may return to earlier domains.

5. **The relationship matters most**. Tools serve connection, not replace it.

### When to Use Journeys

- When someone has a complex, multi-faceted concern
- When you sense they need more than a single-domain solution
- When you have time for deeper exploration
- When they're ready and willing to go deep

### When NOT to Use Journeys

- In crisis (focus on stabilization first)
- When they just need information or a quick answer
- When they're not ready for depth
- When a single tool or domain addresses their need

## Creating New Journeys

To design a new journey:

1. **Identify the transformation**: What changes from start to end?
2. **Map the domains**: Which domains serve this transformation?
3. **Sequence thoughtfully**: What needs to happen first?
4. **Allow flexibility**: Build in branching points
5. **Test with real conversations**: Refine based on experience
