# Deep Human Engagement Domains

This document provides an overview of the 10 new domains designed for deep, meaningful human-level engagement and conversation.

## Overview

While the existing domains (productivity, finance, calendar, etc.) handle functional human needs, these new domains address the **emotional, existential, and relational layers** of human experience that make AI conversation truly meaningful.

---

## The 10 Deep Engagement Domains

### 1. 💞 Relationships & Connection (`/relationships`)

**Purpose**: Nurture relationships, resolve conflicts, deepen human connection.

**Key Tools**:

**Human Need Addressed**: Belonging, love, connection

- **Domains**: relationships, meaning, stories, dreams, self-compassion, vulnerability

### 🌊 River - Grief Companion
- **Domains**: grief, vulnerability, presence, meaning, stories

### 🧘 Zen - Presence & Joy Guide
- **Domains**: presence, play, curiosity, self-compassion

See `CROSS-DOMAIN-JOURNEYS.md` for designed conversation flows that weave through multiple domains for transformative conversations:

1. **Life Review**: stories → relationships → meaning → dreams
2. **Grief Integration**: grief → presence → stories → meaning

4. **Self-Compassion Development**: self-compassion → vulnerability → presence → play
5. **Relationship Deepening**: relationships → vulnerability → self-compassion → meaning

---

## Usage Guidelines

1. **Don't force tools**. Let conversation flow naturally.
2. **Follow the person's lead**. They set the pace and depth.
3. **Combine functional and deep**. A budget conversation might touch meaning.
4. **Respect readiness**. Don't push vulnerability before trust is built.
5. **Presence over tools**. Sometimes just being there matters more.

---

## Technical Details

- All domains use the registry-based tool system
- Each domain exports `getToolDefinitions()` for registration
- Tools use consistent `ToolDefinition` interface
- Domains are registered via `getAllDomainToolDefinitions()`
- Persona manifests specify which domains to include
