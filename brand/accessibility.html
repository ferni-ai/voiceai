Accessibility Guidelines

Ferni is designed to be accessible to everyone. These guidelines ensure our products meet WCAG 2.1 standards and provide an inclusive experience for all users.

WCAG 2.1 AA Compliant Color Contrast: 4.5:1+ Touch Targets: 44px+

Color Contrast

All text must meet WCAG AA minimum contrast ratios: 4.5:1 for normal text and 3:1 for large text (18px+ or 14px+ bold). AAA requires 7:1 and 4.5:1 respectively.

Typography Minimum Sizes

All text must be legible and scalable. Body text should never be smaller than 16px on web or 14px on mobile. Users must be able to zoom to 200% without loss of content.

48px Hero Headlines Page titles
32px Section Titles H1, H2
24px Subsection Headers H3
18px Lead Paragraphs Intro text
16px Body text is the foundation of all readable content. Body (minimum)
14px Captions and helper text - use sparingly Captions only
12px Legal text only - never for primary content Legal / footnotes

Focus States

All interactive elements must have visible focus indicators for keyboard navigation. Focus rings should be at least 3px with sufficient contrast against the background.

Touch Target Sizes

Interactive elements must be large enough to tap accurately. WCAG 2.1 requires a minimum of 44x44 pixels. We recommend 48px for most buttons.

Motion & Animation

Respect user preferences for reduced motion. Use the prefers-reduced-motion media query to disable or minimize animations for users who have enabled this setting.

Standard Motion

animation: bounce 2s ease-in-out infinite;

Reduced Motion

@media (prefers-reduced-motion: reduce) {
  animation: none;
}

Screen Reader Support

All interactive elements and images must have appropriate text alternatives. Use semantic HTML and ARIA labels to ensure content is accessible to screen readers.

Avoid
<div onclick="...">Click</div>
Use
<button aria-label="Submit form">Submit</button>
Avoid
<img src="ferni.png">
Use
<img src="ferni.png" alt="Ferni avatar smiling">
Avoid
<a href="...">Click here</a>
Use
<a href="...">View accessibility guidelines</a>

Accessibility Checklist

Use this checklist when reviewing designs and implementations to ensure accessibility compliance.

Color contrast meets WCAG AA (4.5:1 for text)

Test all text/background combinations using a contrast checker tool.

All images have alt text

Decorative images use empty alt="", informative images describe content.

Interactive elements are keyboard accessible

All buttons, links, and form controls can be accessed with Tab key.

Focus states are visible

Focused elements have a clear visual indicator (3px ring minimum).

Touch targets are at least 44x44 pixels

All tappable elements are large enough for accurate touch input.

Page zooms to 200% without content loss

All content remains readable and functional when zoomed.

Animations respect prefers-reduced-motion

Provide fallbacks for users who prefer minimal motion.

Form fields have labels

Every input has an associated label element or aria-label.

Error messages are clear and helpful

Errors describe the problem and how to fix it, linked to the field.

Content doesn't rely solely on color

Use icons, text, or patterns in addition to color for meaning.
