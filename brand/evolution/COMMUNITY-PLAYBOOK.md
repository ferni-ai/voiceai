|--------|-------------|---------|
| **Growth** | Members supporting each other's growth journeys | Sharing wins, celebrating milestones |
| **Contribution** | Members improving Ferni together | Feature feedback, phrase suggestions |
| **Connection** | Real relationships formed through shared values | Friendships beyond the product |
| **Creation** | Members expressing Ferni values through their work | Fan art, stories, adaptations |

### Moderation Framework

**Community Guidelines:**

1. **Be human** — Treat others with the same warmth Ferni shows you
2. **Respect privacy** — What's shared here stays here
3. **Support growth** — Celebrate wins, support struggles
4. **No judgment** — We all have different journeys
5. **Constructive feedback** — Build up, don't tear down

**Moderation Team:**
- 2 paid community managers
- 5+ volunteer moderators (from Ambassador program)
- Clear escalation path for issues

**Auto-moderation:**
- Slur filter (immediate removal)
- Spam detection
- New member velocity limits

## Ferni Stories Program

### Collecting User Stories

User stories are gold. They provide:
- Social proof for marketing
- Emotional resonance for new users
- Content for PR/media
- Product feedback through narrative

### Story Categories

| Category | Description | Use |
|----------|-------------|-----|
| **Transformation** | "How Ferni changed my life" | Hero testimonials |
| **Moment** | "That one conversation that mattered" | Social content |
| **Daily** | "How I use Ferni every day" | Practical use cases |
| **Persona-specific** | "Why I love Maya" | Persona marketing |
| **Technical** | "Here's what impressed me" | Product validation |

### Story Collection Process

**Opt-in submission:**
```
1. User completes story submission form
2. We review for quality and authenticity
3. We reach out for permission and details
4. User approves final version
5. Story published with appropriate credit
```

**Proactive outreach:**
When we identify users with compelling engagement patterns, we reach out:
- Long-term users (6+ months)
- High engagement (daily users)
- Major milestone achievements
- Unique use cases

### Story Publication Formats

| Format | Length | Platform |
|--------|--------|----------|
| **Quote card** | 1-2 sentences | Instagram, Twitter |
| **Story card** | 1 paragraph | Website, email |
| **Case study** | 500-1000 words | Blog, media |
| **Video testimonial** | 1-3 minutes | YouTube, website |
| **Podcast feature** | 10-20 minutes | Podcast |

## Ambassador Program

### What is a Ferni Ambassador?

Ambassadors are our most passionate users who represent Ferni in their communities. They get:
- Early access to features
- Direct line to the team
- Exclusive merchandise
- Community recognition
- Optional: compensation for specific activities

### Ambassador Responsibilities

| Level | Expectations | Benefits |
|-------|--------------|----------|
| **Seedling** | Active community member, 1 story shared | Ambassador badge, early access |
| **Sprout** | Monthly community contribution, 3 stories | Exclusive merch, feedback sessions |
| **Branch** | Weekly engagement, content creation | Revenue share, event invites |
| **Root** | Community leadership, external representation | All benefits, advisory role |

### Ambassador Activities

**Core activities:**
- Answer questions in Discord
- Share experiences on social media
- Provide product feedback
- Welcome new members

**Optional activities (compensated):**
- Create content (blog posts, videos)
- Host community events
- Represent at conferences
- Recruit new ambassadors

### Ambassador Selection

**Criteria:**
- 3+ months active Ferni use
- Positive community presence
- Authentic enthusiasm (not just seeking perks)
- Alignment with Ferni values

**Application process:**
1. User applies via form
2. Community team reviews application
3. Interview call (15 minutes)
4. Trial period (1 month)
5. Full ambassador status

## Crisis Management

### Types of Crises

| Type | Example | Response |
|------|---------|----------|
| **User safety** | Someone in crisis | Immediate mod intervention, resources |
| **Conflict** | Members fighting | Private de-escalation, public reset |
| **Bad actor** | Troll, spam | Swift ban, community statement if needed |
| **Product failure** | Major bug/outage | Transparent communication, status updates |
| **PR issue** | External criticism | Aligned company response |

### Crisis Response Protocol

1. **Detect** — Mods flag issue
2. **Assess** — Community lead evaluates severity
3. **Respond** — Follow playbook for crisis type
4. **Document** — Record what happened
5. **Review** — Post-crisis analysis

### Escalation Path

```
Member → Moderator → Community Manager → Community Lead → Executive Team
```
